BigDecimal学习

花木凋零成兰

1621字约5分钟

JavaBigDecimal

2024-05-04

《阿里巴巴 Java 开发手册》中提到：“为了避免精度丢失，可以使用 BigDecimal 来进行浮点数的运算”.

因为计算机是二进制的,在表示一个数字时,宽度有限,无限循环的小数存储在计算机时,只能被截断,此时就会发生小数精度丢失的情况, 即浮点数很难用二进制精确表示.

BigDecimal介绍

BigDecimal可以实现对浮点数的运算,且不会造成精度丢失, 当需要浮点数精确运算的结果场景时,应该通过BigDecimal来进行运算.

BigDecimal常用方法

创建BigDecimal

提示

推荐使用BigDecimal(String val)构造方法或使用BigDecimal.valueOf(double val)静态方法创建对象.

因为如果使用BigDecimal(double val)的话,会出现精度的损失,因为构造对象时,参数先是表示为double(此时表示就发生了精度损失),然后再构造BigDecimal对象的.

示例代码如下:

BigDecimal val1 = new BigDecimal(0.009);
System.out.println(val1);   // 0.008999999999999999319988397

BigDecimal val2 = new BigDecimal("0.009");
System.out.println(val2);   // 0.009

BigDecimal val3 = BigDecimal.valueOf(0.009);
System.out.println(val3);   // 0.009

输出后的结果,只有val1出现精度损失,结果为0.0089...

其实在valueOf()方法的内部,也是先将double类型转化为String类型再构造BigDecimal对象,内部代码如下所示:

public static BigDecimal valueOf(double val) {
    // Reminder: a zero double returns '0.0', so we cannot fastpath
    // to use the constant ZERO.  This might be important enough to
    // justify a factory approach, a cache, or a few private
    // constants, later.
    return new BigDecimal(Double.toString(val));
}

加减乘除

  • add() 将两个BigDecimal对象相加
  • subtract() 将两个BigDecimal对象相减
  • multiply() 将两个BigDecimal对象相乘
  • divide(BigDecimal, 保留小数位数, 保留规则) 将两个BigDecimal对象相除, 使用三个参数版本,因为当运算结果为无限循环小数时会抛出异常

相关信息

需要注意,每次BigDecimal进行运算或操作后, 都会产生一个新的BigDecimal对象; 且每次运算的保留规则有8种如下所示:

  • RoundingMode.UP：舍入远离零的舍入模式。在丢弃非零部分之前始终增加数字(始终对非零舍弃部分前面的数字加1)。注意，此舍入模式始终不会减少计算值的大小。
  • RoundingMode.DOWN：接近零的舍入模式。在丢弃某部分之前始终不增加数字(从不对舍弃部分前面的数字加1，即截短)。注意，此舍入模式始终不会增加计算值的大小。
  • RoundingMode.CEILING：接近正无穷大的舍入模式。如果 BigDecimal 为正，则舍入行为与 ROUNDUP 相同;如果为负，则舍入行为与 ROUNDDOWN 相同。注意，此舍入模式始终不会减少计算值。
  • RoundingMode.FLOOR：接近负无穷大的舍入模式。如果 BigDecimal 为正，则舍入行为与 ROUNDDOWN 相同;如果为负，则舍入行为与 ROUNDUP 相同。注意，此舍入模式始终不会增加计算值。
  • RoundingMode.HALF_UP：向“最接近的”数字舍入，如果与两个相邻数字的距离相等，则为向上舍入的舍入模式。如果舍弃部分 >= 0.5，则舍入行为与 ROUND_UP 相同;否则舍入行为与 ROUND_DOWN 相同。注意，这是我们在小学时学过的舍入模式(四舍五入)。
  • RoundingMode.HALF_DOWN：向“最接近的”数字舍入，如果与两个相邻数字的距离相等，则为上舍入的舍入模式。如果舍弃部分 > 0.5，则舍入行为与 ROUND_UP 相同;否则舍入行为与 ROUND_DOWN 相同(五舍六入)。
  • RoundingMode.HALF_EVEN：向“最接近的”数字舍入，如果与两个相邻数字的距离相等，则向相邻的偶数舍入。如果舍弃部分左边的数字为奇数，则舍入行为与 ROUNDHALFUP 相同;如果为偶数，则舍入行为与 ROUNDHALF_DOWN 相同。注意，在重复进行一系列计算时，此舍入模式可以将累加错误减到最小。此舍入模式也称为“银行家舍入法”，主要在美国使用。四舍六入，五分两种情况。如果前一位为奇数，则入位，否则舍去。以下例子为保留小数点1位，那么这种舍入方式下的结果。1.15 ==> 1.2 ,1.25 ==> 1.2
  • RoundingMode.UNNECESSARY：断言请求的操作具有精确的结果，因此不需要舍入。如果对获得精确结果的操作指定此舍入模式，则抛出ArithmeticException。

示例代码如下:

BigDecimal a = new BigDecimal("1.9");
BigDecimal b = new BigDecimal("0.7");
// 相加
BigDecimal add = a.add(b);
System.out.println(add);    // 1.9+0.7=2.6
// 相减
BigDecimal subtract = a.subtract(b);
System.out.println(subtract);   // 1.9-0.7=1.2
// 相乘
BigDecimal multiply = a.multiply(b);
System.out.println(multiply);   // 1.9*0.7=1.33
// 相除 保留两位小数 四舍五入
BigDecimal divide = a.divide(b, 2, RoundingMode.HALF_UP);
System.out.println(divide);     // 1.9/0.7=2.71

设置精度

即设置保留几位小数, 通过setScale()方法,来设置保留几位小数以及对应的保留规则;示例如下:

BigDecimal number = new BigDecimal("0.89319988397");
// 设置保留小数5位以及保留规则为四舍五入 注意此时会产生新的BigDecimal对象
BigDecimal newNumber = number.setScale(5, RoundingMode.HALF_UP);
System.out.println(newNumber);  // 0.89320

大小比较

一般比较大小会涉及到两个方法;equals()和compareTo(), 一般只是比较两个值的话,应该使用compareTo()方法;因为 equals()方法不但会比较值,还会比较精度(scale), 而compareTo()方法则在比较时会忽略精度,示例代码如下:

BigDecimal a = new BigDecimal("3.33");
BigDecimal b = new BigDecimal("3.330");
System.out.println(a.equals(b));    // false
System.out.println(a.compareTo(b)); // 0

输出为字符串

BigDecimal主要有三种转化为字符串的方式:

  • toPlainString(): 不使用任何科学计数法
  • toString(): 在必要的时候使用科学计数法
  • toEngineeringString(): 在必要的时候使用工程计数法,类似于科学计数法,只是指数的幂为3的倍数

即主要是 根据数据结果展示格式不同,采用不同的字符串输出方法, 通常使用比较多的方法为toPlainString(); 示例代码如下:

// 使用valueOf静态方法 创建BigDecimal对象时
BigDecimal number = BigDecimal.valueOf(22345634534124578902.2222228989652632321);
System.out.println(number.toPlainString()); // 22345634534124580000
// 此处默认使用toString()
System.out.println(number); // 2.234563453412458E+19
System.out.println(number.toString());  // 2.234563453412458E+19
System.out.println(number.toEngineeringString());   // 22.34563453412458E+18

::: tips 使用new BigDecimal(String str) 创建BigDecimal对象时, 上述三种输出方式结果均一致,不使用任何科学计数法 :::
