单一职责原则

花木凋零成兰

872字约3分钟

2024-04-15

基本介绍

对类来说, 即一个类应该只负责一项职责;

如类A负责两个不同的职责: 职责1和职责2, 当职责1需求变更而改变A时, 可能造成职责2执行错误, 所以需要将类A的粒度分解为A1, A2

应用实例

以交通工具运行为例;

方案1

使用一个类,来代表交通工具,如下所示:

package top.ytazwc.principle.singleresponsibility;

/**
 * 方案1
 * @author 花木凋零成兰
 * @date 2024/4/15 22:44
 */
public class SingleResponsibility1 {
    public static void main(String[] args) {
        Vehicle vehicle = new Vehicle();
        vehicle.run("摩托车");
        vehicle.run("汽车");
        vehicle.run("飞机");
    }
}

// 交通工具类
class Vehicle {
    public void run(String vehicle) {
        System.out.println(vehicle + " 在公路上运行....");
    }
}

运行代码输出结果如下:

摩托车 在公路上运行....
轮船 在公路上运行....
飞机 在公路上运行....

显然违反了单一职责原则,即负责摩托车在公路上，又负责飞机的运行等.

解决办法: 将该类进行拆分,根据不同的交通工具,分解成不同的类, 详见方案2

方案2

根据方案1,将交通工具类的不同,分解成不同的类,如下所示:

package top.ytazwc.principle.singleresponsibility;

/**
 * 方案2
 * @author 花木凋零成兰
 * @date 2024/4/15 22:47
 */
public class SingleResponsibility2 {
    public static void main(String[] args) {
        RoadVehicle roadVehicle = new RoadVehicle();
        roadVehicle.run("摩托车");

        WaterVehicle waterVehicle = new WaterVehicle();
        roadVehicle.run("轮船");

        AirVehicle airVehicle = new AirVehicle();
        airVehicle.run("飞机");
    }
}
// 方案2分析
class RoadVehicle {
    public void run(String vehicle) {
        System.out.println(vehicle + " 公路运行");
    }
}
class AirVehicle {
    public void run(String vehicle) {
        System.out.println(vehicle + " 天空运行");
    }
}
class WaterVehicle {
    public void run(String vehicle) {
        System.out.println(vehicle + " 水路运行");
    }
}

运行代码结果如下:

摩托车 公路运行
轮船 公路运行
飞机 天空运行

分析:

  1. 遵守单一职责原则
  2. 但是改动过大，即将类分解，同时也需要修改客户端
  3. 改进：直接修改Vehicle类，改动的代码会比较少, 见方案3

方案3

根据方案2中,改动过大,对类分解的同时,还需要修改客户端,考虑在交通工具类中,实现三个对应不同交通工具的方法,如下所示:

package top.ytazwc.principle.singleresponsibility;

/**
 * 方案3
 * @author 花木凋零成兰
 * @date 2024/4/15 22:53
 */
public class SingleResponsibility3 {
    public static void main(String[] args) {
        Vehicle2 vehicle2 = new Vehicle2();
        vehicle2.run("汽车");
        vehicle2.runWater("轮船");
        vehicle2.runAir("飞机");
    }
}
// 方案3分析
class Vehicle2 {
    public void run(String vehicle) {
        System.out.println(vehicle + " 在公路上运行....");
    }

    public void runAir(String vehicle) {
        System.out.println(vehicle + " 在天空上运行....");
    }

    public void runWater(String vehicle) {
        System.out.println(vehicle + " 在水中上运行....");
    }

}

代码运行结果如下:

汽车 在公路上运行....   
轮船 在水中上运行....
飞机 在天空上运行....

分析:

  1. 这种修改方法没有对原来的类做大的修改 只是增加方法
  2. 虽然没有在 类级别上遵守单一职责原则
  3. 但是在方法级别上 仍然遵守单一职责原则

注意事项和细节

  1. 降低类的复杂性,一个类只负责一种职责
  2. 提高类的可读性,可维护性
  3. 降低变更引起的风险
  4. 通常情况下, 我们应当遵守单一职责原则,只有逻辑足够简单,才能在代码级违反单一职责原则,只有类中方法数量足够少,才可以在方法级保持单一职责原则.
