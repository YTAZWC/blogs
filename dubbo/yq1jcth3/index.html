dubbo-service模块

编写Service

在dubbo-service模块中,在top.ytazwc下新建UserService接口类和实现类,内容如下所示:

UserService.class
public interface UserService {
    public String sayHello();
}
UserServiceImpl.class
@Service    // 表示将bean交给Spring管理
public class UserServiceImpl implements UserService {
    @Override
    public String sayHello() {
        return "Hello dubbo";
    }
}

导入配置文件

因为使用的是Spring+SpringMVC进行整合, 所以需要在dubbo-service模块中导入Spring的配置文件,导入后dubbo-service模块目录结构如下:

dubbo-service
│  │  pom.xml
│  │
│  └─src
│      ├─main
│      │  ├─java
│      │  │  └─top
│      │  │      └─ytazwc
│      │  │          └─service
│      │  │              │  UserService.java
│      │  │              │
│      │  │              └─impl
│      │  │                      UserServiceImpl.java
│      │  │
│      │  └─resources
│      │      │  log4j.properties
│      │      │
│      │      └─spring
│      │              applicationContext.xml

配置文件内容如下所示:

log4j.properties
log4j.rootLogger=info, stdout,file
log4j.appender.stdout=org.apache.log4j.ConsoleAppender
log4j.appender.stdout.layout=org.apache.log4j.PatternLayout
log4j.appender.stdout.layout.ConversionPattern=%5p %d %C: %m%n
log4j.appender.file=org.apache.log4j.FileAppender
log4j.appender.file.File=../logs/iask.log
log4j.appender.file.layout=org.apache.log4j.PatternLayout
log4j.appender.file.layout.ConversionPattern=%d{yyyy-MM-dd HH:mm:ss}  %l  %m%n
applicationContext.xml
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
	   xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
	   xmlns:dubbo="http://dubbo.apache.org/schema/dubbo" xmlns:context="http://www.springframework.org/schema/context"
	   xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd
        http://dubbo.apache.org/schema/dubbo http://dubbo.apache.org/schema/dubbo/dubbo.xsd http://www.springframework.org/schema/context https://www.springframework.org/schema/context/spring-context.xsd">

	<!-- 扫描Service包下的bean -->
	<context:component-scan base-package="top.ytazwc.service" />
	
</beans>

dubbo-web模块

添加web.xml文件

因为为Web项目,所以需要在pow文件声明打包为war, 并在main目录下新建webapp目录,然后再新建目录WEB_INF,在该目录下创建web.xml文件,此时dubbo-web模块,文件目录结构如下所示:

dubbo-web
    │  pom.xml
    │
    └─src
        ├─main
        │  ├─java
        │  ├─resources
        │  └─webapp
        │      └─WEB-INF
        │              web.xml
        │
        └─test
            └─java

此时web.xml文件内容如下所示:

点击查看代码
<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xmlns="http://java.sun.com/xml/ns/javaee"
         xsi:schemaLocation="http://java.sun.com/xml/ns/javaee http://java.sun.com/xml/ns/javaee/web-app_2_5.xsd"
         version="2.5">

    <!-- 加载spring容器 -->
    <context-param>
        <param-name>contextConfigLocation</param-name>

        <param-value>classpath:spring/applicationContext*.xml</param-value>
    </context-param>
    <listener>
        <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
    </listener>

    <!-- SpringMVC -->
    <servlet>
        <servlet-name>springmvc</servlet-name>
        <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
        <!-- 指定加载的配置文件 通过参数contextConfigLocation加载 -->
        <init-param>
            <param-name>contextConfigLocation</param-name>
            <param-value>classpath:spring/springmvc.xml</param-value>
        </init-param>
    </servlet>
    <servlet-mapping>
        <servlet-name>springmvc</servlet-name>
        <url-pattern>/</url-pattern>
    </servlet-mapping>
    
    
</web-app>

添加完web.xml文件后, 会出现两个报错,一个是spring/applicationContext*.xml,因为还没有在模块中引入dubbo-service模块,所以会出现爆红, 另一个是spring/springmvc.xml,因此此时还没有在模块中创建该文件,所以会爆红

添加dubbo-service依赖

在pom文件中,添加dubbo-service,内容如下所示, 添加完成后,即可解决上述的第一个报错

<!-- 依赖service模块 -->
<dependency>
    <groupId>top.ytazwc</groupId>
    <artifactId>dubbo-service</artifactId>
    <version>1.0-SNAPSHOT</version>
</dependency>

添加配置文件

即与dubbo-service模块添加配置文件类似, 导入后resources目录结构如下所示:

├─resources
│  log4j.properties
|
└─spring
│          springmvc.xml

文件内容如下所示:

log4j.properties
log4j.rootLogger=info, stdout,file
log4j.appender.stdout=org.apache.log4j.ConsoleAppender
log4j.appender.stdout.layout=org.apache.log4j.PatternLayout
log4j.appender.stdout.layout.ConversionPattern=%5p %d %C: %m%n

log4j.appender.file=org.apache.log4j.FileAppender
log4j.appender.file.File=../logs/iask.log
log4j.appender.file.layout=org.apache.log4j.PatternLayout
log4j.appender.file.layout.ConversionPattern=%d{yyyy-MM-dd HH:mm:ss}  %l  %m%n
springmvc.xml
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:dubbo="http://dubbo.apache.org/schema/dubbo"
       xmlns:mvc="http://www.springframework.org/schema/mvc"
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd
        http://www.springframework.org/schema/mvc http://www.springframework.org/schema/mvc/spring-mvc.xsd
         http://dubbo.apache.org/schema/dubbo http://dubbo.apache.org/schema/dubbo/dubbo.xsd http://www.springframework.org/schema/context https://www.springframework.org/schema/context/spring-context.xsd">

    <mvc:annotation-driven/>
    <context:component-scan base-package="top.ytazwc.controller" />

</beans>

此时,可以解决web.xml文件中的第二个报错, 但是接着会出现top.ytazwc.controller目录报错, 这是因为我们还没有创建该目录, 接着在dubbo-web模块中创建top.ytazwc.controller包即可

编写Controller

在java目录下,新建包top.ytazwc.controller,然后新建UserController类,如下所示:

package top.ytazwc.controller;

@RestController
@RequestMapping("/user")
public class UserController {
    // 注入Service
    @Resource
    private UserService userService;

    @RequestMapping("/sayHello")
    public String sayHello() {
        return userService.sayHello();
    }
}

启动

首先在dubbo-service模块,安装maven的jar包,然后在dubbo-web模块,使用tomcat插件启动项目, 因为在pom文件中,配置的端口为8000,路径为/,所以启动后, 访问localhost:8000/user/sayHello即可

若访问成功,浏览器会显示字符串"Hello dubbo",则说明已成功运行
