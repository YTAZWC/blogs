修改dubbo-service

修改pom文件

设置dubbo-service打包为war; 然后导入tomcat插件,并设置启动端口为9000, 如下所示:

<packaging>war</packaging>

<build>
    <plugins>
        <!--tomcat插件-->
        <plugin>
            <groupId>org.opoo.maven</groupId>
            <artifactId>tomcat9-maven-plugin</artifactId>
            <version>3.0.0</version>
            <configuration>
                <port>9000</port>
                <path>/</path>
            </configuration>
        </plugin>
    </plugins>
</build>

修改Service

即修改提供服务的UserServiceImpl类的注解, 将@Service注解替换为dubbo中的注解,如下所示:

// @Service    // 表示将bean交给Spring管理
@Service    // dubbo中的注解 将该类提供的方法(服务) 对外发布 将访问的地址： ip，端口，路径注册到服务中心
public class UserServiceImpl implements UserService {

    @Override
    public String sayHello() {
        return "Hello dubbo";
    }
}

修改配置文件

同时Spring的配置文件applicationContext.xml中, 需要进行dubbo配置, 其中Spring的bean扫描注释掉, 替换为使用dubbo的注解扫描, 配置内容如下所示:

<!-- dubbo的配置 -->
<!-- 1.配置项目的名称 -->
<dubbo:application name="dubbo-service"/>
<!-- 2.配置注册中心的地址 -->
<dubbo:registry address="zookeeper://ip:端口"/>
<!-- 3.配置dubbo包扫描 -->
<dubbo:annotation package="top.ytazwc.service.impl"/>

创建webapp

因为需要将dubbo-service模块修改为独立可以运行的服务,所以需要配置webapp文件, 创建目录结构如下所示:

└─webapp
    └─WEB-INF
      web.xml

其中web.xml内容如下所示:

<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xmlns="http://java.sun.com/xml/ns/javaee"
         xsi:schemaLocation="http://java.sun.com/xml/ns/javaee http://java.sun.com/xml/ns/javaee/web-app_2_5.xsd"
         version="2.5">
    <!-- spring -->
    <context-param>
        <param-name>contextConfigLocation</param-name>
        <param-value>classpath*:spring/applicationContext*.xml</param-value>
    </context-param>
    <listener>
        <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
    </listener>
</web-app>

启动

配置完成后, 通过tomcat插件启动dubbo-service模块, 若启动成功,则说明成功修改dubbo-service为独立的服务
